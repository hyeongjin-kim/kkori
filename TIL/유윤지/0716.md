## 🧠 상세 내용 정리

### 1. [주제1] 객체 지향(Object-oriented)
: 객체와 속성, 클래스와 멤버, 전체와 부분

- Object
    - 고유 식별자 / 독립 / 일정한 기억 장소 보유
    - state : 조건, 속성 값에 의해 정의
    - Method : 객체가 반응할 수 있는 메시지 집합
- Class
    - 객체들의 집합 / 데이터 추상화 단위
    - Instance : 클래스에 속한 각각 객체
    - Operation : 클래스 동작 / 객체에 적용될 메서드 정의
- Abstraction
    - 복합한 것에서 핵심만 뽑아 단순화하는 것
      → 불필요한 세부사항 숨기기, 중요한 것만 드러내기
- Encapsulation
    - 데이터 - 데이터 처리 함수 묶음
    - 정보 은닉 → 외부 접근 제한
    - 결합도 낮 / 재사용 용이 / 인터페이스 단순 / 오류 파급효과 낮
- Inheritance
    - 상위 속성과 연산을 하위 클래스가 물려받음
    - 다중 상속 : 단일 클래스 2개 이상의 상위 클래스로부터 상속
- Polymorphism
    - 1 메시지에 각 객체 별 고유 특성에 따라 여러 형태 응답

> Overriding vs Overloading
>
- Overriding : 상속 받은 메서드 하위 클래스에 재정의
  → 메서드 이름 / 매개변수 / 반환 타입 동일
- Overloading : 메서드 이름 동일 , 매개변수 개수 / 타입 다르게 지정
