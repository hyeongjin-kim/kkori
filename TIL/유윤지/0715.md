## 🧠 상세 내용 정리

### 1. [주제1] 보안 관리 방법

### HTTPS 필수 이유

1. HTTP 한계/위험
    - 데이터를 암호화 하지 않고 주고 받는 방식
    - 스니핑(누군가 네트워크를 중간에서 엿보는 것) 시, 로그인 정보, 카드번호, 개인정보 등이 그대로 노출될 가능성 존재

   e.g.) 카페 와이파이 등 공개 네트워크 도청, …

2. HTTPS 역할
   HyperText Transfer Protocol Secure
    - HTTP + 암호화(SSL/TLS)
    - 모든 데이터 암호화 = 중간에서 훔쳐봐도 내용 알 수 X
    - 로그인 정보, 세션/토큰, 개인정보 안전 보호
    - 서버의 진짜 신원(인증서) 검색해 피싱 사이트 구분 가능
3. HTTPS 동작 흐름
    - 클라이언트가 서버에 접속할 때, 서버는 공개키와 인증서 전달
    - 브라우저는 인증서가 신뢰할 수 있는 기관(CA)에서 발급된 것인지 확인
    - 브라우저와 서버는 암호화 통신에 사용할 비밀키(세션키) 안전하게 교환
    - 모든 데이터는 해당 키로 암호화 되어 오감

> [정리]
>
> - **데이터 도청 방지**: 중간에서 데이터를 훔쳐볼 수 없음
> - **위조 사이트 방지**: 인증서로 서버의 진짜 신원 검증
> - **데이터 위변조 방지**: 데이터가 중간에서 바뀌는 것 방지
> - **신뢰성 향상**: 사용자 브라우저에 자물쇠 표시가 떠서 신뢰감

### 2. [주제2] Hijacking 방지 방법

### Session Hijacking 방지 방법

- Secure 옵션 : HTTPS 연결에서만 쿠키 전송
- HttpOnly 옵션 : JS로 쿠키 읽을 수 없음
- SameSite 옵션 : 타 사이트에서 쿠키가 자동 전송되는 것 방지
- 세션 만료 : 일정 시간 동안 사용하지 않으면 세션 자동 만료 → 탈취된 세션 오래 사용 X
- 정기적 Session ID 재발급 : 로그인/중요 정보 변경 시, Session ID 새로 발급해 세션 고정 공격 방지

### Token(JWT) Hijacking 방지 방법

- HTTPS 사용 : 토큰 네트워크 노출 방지
- 쿠키 저장 시 Secure/HttpOnly/SameSite 옵션 : 세션 방식과 동일하게 쿠키에 저장한다면 위 옵션 사용 필수
- LocalStorage/SessionStorage 저장 시 : XSS 공격 취약하니, 브라우저 저장소에 직접 저장하는 방식을 최소화하고, 엄격한 XSS 방어 코드 적용
- 짧은 토큰 만료 시간 : 토큰 탈취되어도 오래 사용할 수 없도록 만료 시간 짧게 설정
- Refresh Token 분리 관리 : 장기 인증 필요 시, 짧은 만료의 Access Token과 별도의 Refresh Token 사용해 Refresh Token은 HttpOnly 쿠키에만 저장
- 로그아웃/탈취 대응 : 서버에서 토큰 블랙리스트 관리 / Refresh Token을 서버에 저장해 탈취 시 무효화

| **구분** | **세션 기반 인증**               | **토큰(JWT) 기반 인증**            |
|--------|----------------------------|------------------------------|
| 저장 위치  | 서버(세션), 브라우저(쿠키)           | 클라이언트(쿠키/로컬스토리지 등)           |
| 쿠키 옵션  | Secure, HttpOnly, SameSite | 쿠키 저장 시 동일 옵션 적용             |
| 만료 관리  | 세션 타임아웃, 세션 ID 재발급         | 토큰 만료시간 짧게, Refresh Token 사용 |
| 탈취 대응  | 세션 무효화, 재로그인 유도            | 토큰 블랙리스트, Refresh Token 무효화  |
| 주의점    | 세션 동기화, 서버 확장성             | 토큰 저장 위치(XSS), 만료 관리         |

### 요약

- HTTPS는 모든 인증 방식 기본 : 도청 막기 위해 사용 필수
- 쿠키 안전하게 설정 : Secure, HttpOnly, SameSite 옵션 필수
- 세션/토큰 모두 만료 관리 : 오래된 인증 정보는 위험 → 자동 만료
- 토큰은 브라우저 저장소 직접 저장 시 XSS 주의 : 가능하면 HttpOnly 쿠키 사용
- 로그아웃/탈취 대응 체계 필요 : 세션/토큰 유출 시 바로 무효화 할 수 있어야 함
