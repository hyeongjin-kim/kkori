### 1. [주제1] SOLID

- SRP(Single Responsibility Principle)

  → 모든 클래스/객체 하나의 책임

  → 완전한 캡슐화

- OCP(Open Closed Principle)

  → 확장 Open

  → 수정 Close

- LSP(Liskov Substitution Principle)

  → 상위 클래스 행동 규약을 하위가 위반 X

- ISP(Interface Segregation Principle)

  → 클라이언트가 비사용 메서드 의존 X

- DIP(Dependency Inversion Principle)

  → 의존 관계 수립 시 변화 어려운 것(추상성 높은 상위 클래스)에 의존

### OSI 7계층

### TCP/IP 4계층

* 데이터를 보내고 받는 과정

- Application(응용)
  응용 프로그램이 OSI 환경에 접속 가능한 서비스
  e.g.) 웹(HTTP), 이메일(SMTP, POP), 파일 전송(FTP), 원격접속(Talent), 도메인(DNS)
- Presentation(표현)
  응용 - 세션 간 코드/데이터 변환
  데이터 암호화/압축
  e.g.) 이미지, 영상, 문자 인코딩/디코딩, SSL 암호화

- Session(세션)
  컴퓨터 간 세션/연결을 생성/유지/종료해 적절한 통신 상태 유지
  e.g.) 원격 접속, 화상 회의 연결 관리

- 응용
    - Telnet/FTP
    - HTTP/POP
    - SMTP
    - DHCP/SNMP
    - DNS

---

- Transport(전송)
  End-to-End(종단) 간 투명한 데이터 전송/전달
  주소 설정/다중화/요류 제어/흐름 제어 수행
  e.g.) TCP, UDP

- 전송
    - TCP/UDP

---

- Network(네트워크)
  개방 시스템 간 네트워크 연결 관리/데이터 교환
  경로 설정(Routing), 트래픽 제어, 패킷 정보 전송
  e.g.) IP 주소, 라우터

- 인터넷
    - IP/ICMP
    - ARP/RARP

---

- Data Link(데이터 링크)
  송/수신 간 속도 차이 해결 위한 흐름 제어 기능
  프레임 동기화/오류 제어 기능/노드 간 프레임 전송 → MAC(물리적 주소) 결정
  e.g.) 스위치, 이더넷, MAC 주소

- Physical(물리)
  전송에 필요한 기계적, 전기적, 기능적 특성 정의
  데이터를 0과 1의 전기적 신호로 변환
  e.g.) 랜선, 와이파이, 허브, …

- 네트워크 액세스
    - HDLC/PPP
    - LLC


---

> OSI 7계층의 흐름
>
1. 사용자(응용 계층) : 웹 브라우저로 네이버에 접속(HTTP)
2. 표현 계층 : 웹 페이지 데이터 압축/암호화
3. 세션 계층 : 네이버와 연결 세션 생성
4. 전송 계층 : 데이터가 순서대로, 빠짐없이 전달되도록 관리(TCP)
5. 데이터 링크 계층 : 네트워크 내에서 목적지 컴퓨터(MAC)까지 데이터 전달
6. 물리 계층 : 0과 1의 신호로 실제 전송
