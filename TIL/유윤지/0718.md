## 🧠 상세 내용 정리

### 1. [주제1] JAVA
- `1.자바란 무엇인가요?`

  자바는 객체지향 언어로 추상화, 다형성, 상속, 캡슐화(APIE) 같은 개념을 기반으로 구조적인 설계를 지원합니다. JVM 위에서 동작하기 때문에 운영체제에 독립적인 ‘Write Once, Run Anywhere’ 이라는 특징이 있습니다. 또한 멀티쓰레드 지원과 자동 메모리 관리로 서버 개발에 적합합니다.

    - `자바가 현대 프로그래밍에서 많이 쓰이는 이유는 무엇일까요?`

      자바는 수십 년간 축적된 라이브러리와 프레임워크 덕분에 대규모 시스템에서 신뢰 받고 있습니다.

    - `Node와 비교했을 때 Java가 가지는 장점과 단점을 말해주세요`

      자바는 정적 타입 기반이라 컴파일 타임에 오류를 잡을 수 있고, 구조적인 대규모 백엔드에 적합합니다.

      반면 노드js는 비동기 처리와 빠른 개발 속도에서 장점이 있지만, 런타임 에러가 자주 발생할 수 있어 관리가 어렵습니다.

      안정성과 확장성을 고려해야 하는 서비스라면 자바를 선호합니다.

    - `자바는 비동기 처리에 있어서 노드에 비해 안좋나요?`

      노드의 경우 이벤트 루프 기반의 논블로킹 구조로 비동기에 강점을 보입니다. 반면 자바는 기술적으로 WebFlux나 CompletableFuture로 고성능 비동기 처리가 가능하지만, 기본 구조가 쓰레드 기반이라 이벤트 루프 방식의 노드보다 구현 복잡도가 높습니다.

      다만, 자바는 안정성과 예외 처리에 강점이 있습니다.
