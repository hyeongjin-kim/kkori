## 📌 오늘 학습한 내용 요약

- `자바스크립트는 왜 싱글 쓰레드인가요?`

  자바스크립트는 원래 브라우저의 UI를 조작하기 위한 언어로 설계되었기 때문에 여러 쓰레드가 동시에 DOM을 수정하면서 생기는 충돌을 피해야 했습니다. 그래서 동시성 문제가 없도록 싱글 쓰레드 기반으로 설계되었습니다.

  - `자바스크립트는 왜 멀티태스킹처럼 보이나요?`
    자바스크립트는 싱글 쓰레드이지만, 이벤트 루프와 백그라운드 쓰레드를 함께 사용해 비동기 작업을 병렬적으로 처리하는 것처럼 동작합니다.
  - `이벤트 루프가 무엇인가요?`
    자바스크립트에서 비동기 작업을 처리하는 핵심 메커니즘입니다.
    비동기 처리 요청이 들어오면 콜 스택에서 WebAPI에 요청하고 완료된 비동기 처리는 태스크 큐에 저장됩니다.
    태스크 큐와 콜 스택을 감시하면서 콜 스택이 비면 큐에 대기중인 작업을 하나씩 꺼내 실행합니다.
    - `Task queue와 micro Task queue는 어떻게 다른가요?`
      Task queue에는 setTimeout과 setInterval, DOM 이벤트, fetch 콜백이 등록됩니다.
      micro Task queue에는 Promise.then, process.nextTick(micro task queue에서 가장 먼저 실행됨), MutationObserver가 콜백에 등록됩니다.

- `커널이 뭔가요?`

  커널은 CPU, 메모리, 디스크 같은 하드웨어 자원을 직접 제어하고, 사용자 프로그램이 요청한 작업을 안정적이고 효율적을 수행할 수 있도록 관리하는 핵심 소프트웨어입니다.

  - `커널 쓰레드가 뭔가요?`
    커널이 직접 생성하고 관리하는 실행 단위입니다. 운영체제는 커널쓰레드를 이용해서 스케줄링, 문맥 교환, 동기화 등을 처리합니다.
  - `어플리케이션의 쓰레드와 커널 쓰레드는 몇 대 몇으로 연결되나요?`
    운영체제의 쓰레드 모델에 따라 다르지만, 대표적으로 1:1, N:1, N:M 모델이 있습니다. 대부분의 현대 운영체제는 1:1모델을 사용해 어플리케이션 쓰레드 하나마다 커널 쓰레드가 하나씩 연결됩니다.
  - `쓰레드는 무조건 많은 것이 좋은가요?`
    쓰레드가 늘어날 수록 병렬성을 확보하게 되면서 성능이 향상되지만, 어느 순간에 도달하면 프로세스가 작업을 분할 할 수 있는 한계에 도달해 성능이 향상되지 않고 오히려 context change나 메모리 사용량 증가로 성능이 저하됩니다. 따라서 적절한 수의 쓰레드가 있는게 좋습니다.
  - `동시성과 병렬성은 어떻게 다른가요?`
    동시성(Concurrent)는 작업들이 실제로는 순차적으로 수행되지만, 빠르게 스위칭해서 “동시에 수행되는 것처럼” 보이는 것을 의미합니다.
    병렬성(Parallelism)은 작업들이 실제로 동시에 실행되는 것을 말합니다.

- `폰노이만 구조가 뭔가요?`
  현대 컴퓨터의 기본 설계 방식으로, 프로그램과 데이터를 동일한 메모리에 저장하고 이를 순차적으로 처리하는 구조를 말합니다.
  CPU(중앙처리장치), 입력 장치, 기억 장치, 출력 장치로 구성됩니다.
  - `하버드 구조가 뭔가요?`
    폰노이만 구조와 달리 코드와 데이터가 분리된 메모리 공간에 저장되는 컴퓨터구조입니다.
    명령어 메모리와 데이터 메모리를 분리해서 동시에 접근이 가능해 처리 속도와 병렬성이 더 뛰어납니다.
