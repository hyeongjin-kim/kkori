## 🧠 상세 내용 정리

- 
    
    `This` 는 실행 문맥에 따라서 참조 대상이 달라지는 동적 바인딩입니다.
    
    기본적으로 전역에서는 `window` , 메서드 안에서는 메서드를 호출한 객체를 참조합니다.
    
    이벤트 핸들러나 콜백에서는 별로도 바인딩을 해야 할 수 있습니다.
    
    - call, bind, apply의 차이점 무엇인가요?
        
        `call, applly, bind` 는 모두 `this` 를 원하는 객체로 바꿔주는 함수입니다.
        
    - 화살표 함수에서의 `this` 는 어떻게 작동하나요?
        
        화살표 함수는 자신만의 `this` 를 가지지 않고, 상위 스코프의 `this` 를 그대로 사용합니다.
        
        그래서 콜백에서 유용하게 사용할 수 있습니다.
        
- `클로저가 무엇인가요?`
    
    클로저는 함수가 자신이 선언될 당시의 렉시컬 환경을 기억하고, 그 외부 변수에 접근할 수 있는 구조입니다.
    
    - `클로저는 언제 사용하나요?`
        
        클로저는 데이터를 은닉하거나, 상태 유지가 필요할 때 클로저를 사용할 수 있습니다.
        
        ```jsx
        function createCounter() {
        	let count = 0; -> count는 외부에서 접근할 수 없음.
        	return function () {
        		return ++count;
        	};
        }
        const counter = createCounter();
        counter(); // 1
        counter(); // 2
        ```
        
    - `클로저의 단점은 무엇인가요?`
        
        메모리 누수 위험이 있습니다. 불필요한 외부 변수를 참조한 채 함수가 오래 살아있으면 가비지 컬렉션이 불가능해집니다
        
- `함수형 프로그래밍이 무엇인가요?`
    
    함수형 프로그래밍은 상태 변경과 그로 인한 부작용을 최소화하기 위해 순수 함수를 중심으로 구성하는 프로그래밍 패러다임입니다.
    
    - `함수형 프로그래밍을 하는 이유가 무엇인가요?`
        
        함수 실행이 외부 상태를 변경하거나, 외부 상태에 의존하면 예측이 어렵고 테스트가 어려워지는 문제가 있습니다. 
        
        함수형 프로그래밍은 이를 방지하기 위해 동일한 입력에 항상 동일한 출력을 내고, 외부 상태를 변경하지 않는 순수 함수로 구성합니다.
        
    - `순수 함수는 무엇인가요?`
        
        동일한 입력에 항상 동일한 출력을 내고, 외부 상태를 변경하지 않는 함수를 순수 함수라고 합니다.
        
    - `커링이 무엇인가요?`
        
        커링은 함수가 여러 인자를 한 번에 받는 대신, 하나씩 인자를 받아 그때마다 새로운 함수를 반환하는 방식입니다.
        
- `일급 객체가 무엇인가요?`
    
    일급 객체는 값처럼 변수에 할당할 수 있고, 함수의 인자로 넘기거나 반환값으로도 쓸 수 있는 것을 말합니다.
    
    자바스크립트의 함수는 일급 객체라서 고차 함수나 콜백 패턴 구현이 가능합니다.
    
    - `객체와 일급 객체의 차이는 무엇인가요?`
        
        객체는 여러 값을 하나로 묶은 자료구조이고, 일급 객체는 값처럼 변수에 할당, 인자로 전달, 반환이 가능한 지위입니다. 객체는 자료형이고 일급 객체는 역할의 개념입니다.
        
    - `배열은 객체인가요?`
        
        자바스크립트의 배열은 다른 언어에서 연속된 메모리에 변수를 저장하는 배열을 흉내낸 유사 배열 객체입니다. `index` 를 key로 사용하여 객체로 값을 저장합니다.
