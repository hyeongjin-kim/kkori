## 🧠 상세 내용 정리

### 1. JSON이란?

- 정의 : 데이터를 구조화하여 표현하기 위해 만들어진
        텍스트 기반의 가벼운 데이터 교환 포맷
- 특징 : 텍스트 기반, 키-값 쌍 구조, 언어 비종속적, 경량성, 상호운용성, 확장용이성
- 장점
    (1) 가독성이 좋음
    (2) 유연한 구조(필드가 없는 경우 생략 가능)
    (3) XML 대비 구조가 단순하고 데이터 양이 적음
- 단점
    (1) 스키마가 없어서 예측하지 못한 오류 발생 가능
    (2) 공식적으로 주석을 지원하지 않아서 설명 추가가 어려움
    (3) 데이터 타입이 제한돼있어서 날짜, 이진 데이터 등은 별도 규칙 인코딩 필요

### 2. JWT(JSON Web Token)란?

- 정의 : JSON 기반으로 이루어진 인증
        및 정보 전달에 사용되는 토큰
- 특징 : 무상태성으로 별도 서버 세션 저장소 없이 인증 가능
- 장점
    (1) 다양한 환경 지원
    (2) 전송 및 저장 용이
    (3) 빠른 인증 가능
- 단점
    (1) 즉시 폐기 불가능
    (2) 정보 필드 수에 따른 크기 증가
    (3) Key 유출 시 위험 증가

### 3. Spring Security란?

- 정의 : 스프링 기반의 인증, 인가, 보안 관련 필터
        및 정책을 제공하는 프레임웤
- 특징 : Spring 생태계와 통합
- 장점
    (1) 강력한 보안 기능
    (2) 유연한 커스터마이징 가능
    (3) 확장성이 좋음
- 단점
    (1) 초심자에겐 학습 난이도가 높음
    (2) 시스템 구조가 복잡함

### 4. Refresh Token이란?

- 정의 : 만료된 Access Token을 클라이언트가
        재발급하기 위해 사용하는 별도의 토큰
- 특징 : Access Token보다 만료 시간이 긺
- 장점
    (1) 보안성 높음
    (2) 사용자 경험 개선(재로그인 필요 최소화 등)
    (3) 유연한 세션 관리 가능
- 단점
    (1) 서버 저장 및 관리 필요
    (2) 보안 사고 시 위험성 높음
    (3) 구현이 복잡함

### 5. Refresh Token Rotation이란?

- 정의 : Refresh Token을 재사용할 때마다
        새로운 토큰을 발급하고, 이전 토큰은 즉시 폐기하는 보안 기법
- 특징 : 단일 유효성
- 장점
    (1) 공격 유효 시간 최소화(토큰의 무효화)
    (2) 보안 이벤트 탐지 후 알림
    (3) 지속적 세션 유지
- 단점
    (1) 클라이언트 간 동기화 필요
    (2) 구현 난이도가 높음
    (3) 네트워크 문제 발생 시 재인증 요구
