## 🧠 상세 내용 정리

1. 그렇다면, 매 요청마다 Resource Server(예: 카카오, 구글)에 토큰 유효성 검증을 요청해야 하나?
실제 상황
내 서버는 토큰이 opaque token이라 토큰 내부를 해석할 수 없고,
따라서 매 요청 시마다 카카오와 같은 Resource Server에 "이 토큰 유효한가?" 를 API로 확인해야 한다.

하지만, 매 요청마다 검증 요청을 보내면 네트워크 지연과 부하가 커지므로 실제 서비스에서는 검증 결과를 일정 시간 캐싱하거나, 민감한 요청에 한해서만 검증하는 전략을 사용한다.

2. Access Token을 DB에 저장하고 자체 검증하면 안 될까?
예를 들어, 발급받은 Access Token을 내 서버의 DB에 저장하고,
클라이언트가 보낼 때마다 이 토큰이 존재하는지만 확인하면 되지 않을까?

문제점
이 방식은 보안에 매우 취약하다.

Access Token은 외부 서비스(예: 카카오)에서 관리하는 자원이다.
사용자가 탈퇴하거나, 토큰이 강제 만료되었을 경우
→ 내 DB에는 여전히 유효하다고 판단하여 인증을 허용해버릴 수 있다.
즉, 외부 상태를 반영하지 못한다는 점에서 신뢰할 수 없다.

3. Access Token의 유효 시간이 짧은데, 그럼 자주 로그인해야 하나?
Access Token이 만료되면 사용자는 다시 로그인해야 한다.
이 불편을 줄이기 위해 Refresh Token이라는 개념이 등장했다.

Access Token은 짧게 (예: 1시간)
Refresh Token은 길게 (예: 30일)
Refresh Token을 통해 새로운 Access Token을 발급받을 수 있다. Refresh Token은 아래에서 자세히 다룬다.

4. Refresh Token도 탈취되면 위험하지 않을까?
Refresh Token이 탈취되면 오히려 Access Token보다 더 위험하다.
왜냐하면, 이걸로 Access Token을 무제한으로 재발급받을 수 있기 때문이다.

이를 방지하기 위해 Refresh Token Rotation 기법을 사용한다.
→ Refresh Token을 사용할 때마다 새로운 Refresh Token으로 교체한다.

이 방식이라면, 만약 누군가 이전 토큰을 탈취해 사용하려 하면
이미 서버에는 새로운 Refresh Token만 등록되어 있기 때문에 재사용을 막을 수 있다.

5. Access Token이 외부에서 강제로 만료되었을 경우, 내 서버는 어떻게 알 수 있을까?
알 수 없다.
Access Token을 자체적으로 저장해 검증하는 방식이라면, 카카오 측에서 이 토큰을 강제 만료시켜도 내 서버는 인지할 수 없다.

이를 해결하기 위해서는, 주기적으로 또는 중요한 요청마다 Resource Server에 토큰 유효성 검증 요청을 보내야 한다.
물론 모든 요청에서 검증하는 것은 오버헤드가 크기 때문에 민감한 API 호출 시에만 선택적으로 검증하는 방식을 사용할 수 있다.

그렇다면, 매 요청마다 Resource Server에 토큰 유효성 검증을 요청해야 하나?

Access Token을 DB에 저장하고 자체 검증하면 안 될까?

Access Token이 외부에서 강제로 만료되었을 경우, 내 서버는 어떻게 알 수 있을까?

이 문제의 공통점은, 토큰의 검증이 외부 서비스에 의존 된다는 것이다.

이러한 문제를, 보통 클라이언트 자체의 JWT토큰을 발행하고 사용하는 것으로 해결한다.

기존 흐름과 동일하게, Authorization Code를 통해 ID Token과 AccessToken을 발급받는다.
ID Token을 통해 클라이언트 자체의 JWT토큰을 발행한다.
사용자는 이후 요청부터 자체 JWT를 Authorization 헤더에 담아 보낸다.
클라이언트는 이 JWT만 검증해도 인증 여부를 판단할 수 있다 (외부에 의존 X).
우선 문제점 1, 2의 공통 문제는 토큰의 검증에 발생하는 네트워크 부하이다.
이는 자체 JWT 토큰을 사용하는 것으로 해결할 수 있다.

문제 1, 3의 경우, 외부 서비스에서 발행한 Access Token이 필요한가? 에 따라 나눌 수 있다.
외부 API 호출이 필요할 경우에는 여전히 외부 서비스의 Access Token을 사용해야 한다.

카카오톡 친구 목록 가져오기, 구글 캘린더 읽기 등
이 경우 Access Token은 서버에 암호화해서 저장 후 사용 시점에만 활용하는 전략이 필요하다.
