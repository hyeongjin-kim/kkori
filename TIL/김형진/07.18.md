## 🧠 상세 내용 정리

### 1. [주제1] 예: 자바스크립트는 왜 싱글 쓰레드인가?

- 자바스크립트가 생성될 당시에는 멀티 쓰레딩이 일반적이지 않았습니다.
- 자바스크립트는 브라우저의 UI를 조작하기 위해 설계된 언어였습니다.
- DOM에 여러 쓰레드가 동시에 접근해서 충돌이 발생하는 것을 방지하기 위해 동시성의 문제가
  없는 싱글 쓰레드 방식을 채택했습니다.
- 자바스크립트는 왜 싱글쓰레드인데 멀티테스킹처럼 보이나요?
- 이벤트 루프를 통해 비동기 작업을 병렬적으로 처리하기 때문입니다.
- 이벤트 루프는 무엇인가요?
- 이벤트 루프는 자바스크립트에서 비동기 처리를 관리하는 핵심 메커니즘입니다.
- 콜 스텍에서 비동기 요청이 들어오면 이를 Web API로
  보내고, 완료되면 이벤트 루프가 task queue에 담습니다.
  이후 콜스택이 비게 되면 이벤트 루프는 task queue에서 비동기 작업을 하나 꺼내 콜스택에 등록하여 처리합니다.
- Task queue와 micro Task queue는 어떻게 다른가요?
- micro Task queue는 Task queue보다 우선 순위가 높은 작업을 담습니다.
- Task queue에는 setTimeout, setInterval 등의 콜백이 담깁니다.
- micro Task queue에는 Promise.then, process.nextTick이 담깁니다.
