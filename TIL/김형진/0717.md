## 🧠 상세 내용 정리

### 1. [주제1] 예: Promise란?

- 정의 : 비동기 처리의 완료를 나타내는 객체 then, catch, finally를 통해 비동기 흐름을 체이닝으로 명확하게 관리함
- Promise가 등장한 이유: 콜백으로 비동기를 처리할 때는 콜백이 이어질수록
  depth가 늘어나고 코드가 복잡해져 가독성이 떨어지는 콜백 지옥이 발생했기 때문

  Promise는 체이닝을 통해 일정한 깊이를 유지함

- Promise의 단점 : 외부에서는 비동기 처리의 완료
  시점을 알 수 없기 때문에 콜백에서 이를 처리해야함
  하지만 체이닝 방식을 이용하기 때문에 콜백이 계속되면 코드가 점점 길어져 가독성이 떨어짐

- async, await란: 비동기 흐름을 동기적으로
  표현하기 위해 만들어진 문법
  await가 비동기 처리의 종료 시점까지 대기하기 때문에 비동기를 동기처럼 사용할 수 있고, 체이닝을 활용하지 않기 때문에 가독성도 좋음

### 2. [주제2] HTTP란?

- 정의 : 문서나 데이터를 전송하기 위해 만들어진
  통신 규약
- HTTP의 특징 :
  비연결성 : 요청과 응답이 끝나면 연결이 끊어짐
  무상태성 : 서버는 이전 응답의 결과를 기억하지 않음
- HTML이 무엇인지 : 문서나 데이터를 표현하기 위해
  만들어진 마크업 언어
- HTTP 0.9의 특징 :
  GET만 존재했음
  문서만 전달 가능
- HTTP 1.0의 특징
  연결을 유지하는 기능이 생김(옵션)
  상태 코드가 생김

- HTTP 1.1의 특징
  응답이 완료됐을 때 연결을 종료하지 않는
  keep alive가 표준화됨
  pipelinig으로 요청에 대한 HOLB(head of line blocking)을 해결함
- HTTP 2.0의 특징
  패킷을 프레임으로 나누고 이를 바이너리로 인코딩해 압축하는 바이너리 프레이밍
  연속적으로 데이터를 받는 스트림을 이용해 여러 요청을 한번에 처리할 수 있는 멀티 플랙싱
